Reflected XSS Vulnerability

High Severity Detected in search.js:42

A reflected Cross-Site Scripting (XSS) vulnerability was detected in the search functionality. This vulnerability allows attackers to inject malicious scripts that execute in users' browsers when they click on specially crafted links.

Steps to Reproduce

  1. Navigate to the search page
  2. Enter the payload <script>alert(document.cookie)</script> in the search field
  3. Submit the search form
  4. The script executes in the browser, displaying the user's cookies

Impact

This vulnerability can lead to:

  • Session hijacking
  • Data theft
  • Unauthorized actions performed on behalf of the user
  • Defacement of the website

Vulnerable Code

// search.js:42
function displaySearchResults(query) {
const resultsDiv = document.getElementById('searchResults');
resultsDiv.innerHTML = `<h2>Search Results for: ${query}</h2>`;
// Rest of the function...
}

Recommended Fix

Implement proper input sanitization and output encoding to prevent XSS attacks: 

// Fixed version
function displaySearchResults(query) {
const resultsDiv = document.getElementById('searchResults');
const sanitizedQuery = escapeHTML(query); // Use a proper sanitization function
resultsDiv.innerHTML = `<h2>Search Results for: ${sanitizedQuery}</h2>`;
// Rest of the function...
}
function escapeHTML(str) {
return str.replace(/[&<>"']/g, function(match) {
return {
'&': '&',
'<': '<',
'>': '>',
'"': '"',
"'": '''
}[match];
});
}
